# Security Policy

## Reporting Security Issues

If you discover a security issue or believe sensitive material has been accidentally included in the repository, do not post full details publicly in an issue.

Instead, report it privately to the project stewards or designated maintainers.

## Examples of Sensitive Issues

Sensitive issues may include:

- accidentally committed credentials
- authentication or configuration secrets
- private server access information
- exposed private operational files
- sensitive vulnerabilities that should not be publicly disclosed immediately

## Public Issues

Do not post the following in public issues:

- passwords
- tokens
- private configuration values
- database credentials
- live operational access details
- sensitive exploit instructions

## Response Approach

When a sensitive issue is reported, maintainers and Community Stewards may:

- remove or rotate exposed secrets
- restrict access where needed
- clean repository history if necessary
- document and remediate the issue privately before broader disclosure

## Scope

This repository is a public/community-maintained project repository. Live server operations, private backups, and production environments may involve separate security considerations not fully represented here.

## Responsible Reporting

Please report issues responsibly and privately where appropriate.
