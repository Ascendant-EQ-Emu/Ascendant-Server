# Ascendant Server FAQ

## What is this repository?

This repository is the community-maintained codebase and project snapshot for Ascendant Server.

It exists to preserve the Ascendant server project in an open, maintainable form so that the project does not depend entirely on any one operator, machine, or hosting arrangement.

It includes the code, server-side project files, and related materials that the community may use to understand, preserve, and continue the project.

## How is this related to the live Ascendant server?

This repository is directly related to the Ascendant project and to the live Ascendant server historically and practically.

Straps was the original maintainer of Ascendant and continues to host a freely available live instance of Ascendant for the community.

There is also an active Discord community for that server:

- https://discord.gg/g54hFzAjJp

At the same time, this repository is intended to be a community-owned open project. It is related to the Ascendant server project, but it is not treated as private intellectual property formally bound to one specific live server instance forever.

## Is there an official Ascendant Server?

Yes.

The **official Ascendant Server** refers to the live server instance recognized by the project stewards and operators as the primary live Ascendant server.

At present, Straps continues to host a freely available live Ascendant instance, and that server is the clearest reference point for the official live Ascendant experience unless the project stewards state otherwise.

## So what is the difference between this repository and the official live server?

The repository is the shared community codebase and project snapshot.

The official live server is an actively operated deployment of that codebase, with its own live data, operational environment, hosting, and community operations.

In simple terms:

- the repository is the project
- the live server is one running instance of the project

## Does this mean anyone can spin up an Ascendant server?

Anyone can use the repository to create a compatible server instance based on the public project materials, subject to the repository license and any applicable legal obligations.

However, that does **not** automatically make every such instance an official Ascendant server.

A separate operator may be able to run a compatible instance, fork, test environment, or community-derived deployment. That is part of what an open community project allows.

## If anyone can spin up a compatible instance, what makes one server “official”?

A server is not “official” simply because it uses this repository.

An official Ascendant server is one recognized by the project’s Community Stewards and current project leadership as the primary live Ascendant instance.

Open availability of the codebase does not eliminate the distinction between:

- a compatible instance
- a fork
- a test environment
- the official live Ascendant server

## Is this repository the same thing as the live Ascendant server?

No.

This repository is a curated, community-maintained project snapshot. It is related to the Ascendant project and live server, but it is not identical to any one live private production environment.

It is meant to preserve the project responsibly, not expose every private operational detail of a running server.

## Is this repository private IP belonging to one person or one server?

No.

This repository is intended to be a community-owned open project.

It is related to the Ascendant project and the live Ascendant server history, but it is not treated as private intellectual property formally and permanently bound to one specific server operator.

That is one of the main reasons this repository exists: to make the project preservable and maintainable beyond any single host or operator.

## Does this repository include player data?

No.

This repository does **not** include live player accounts, character data, private runtime secrets, or other sensitive production data from a live Ascendant server.

That material remains private.

## Are backups of the live server database public?

No.

Live database backups are not public repository contents.

In some cases, backups may be preserved under community stewardship with restricted access and privacy protections in place, but that is different from making them public.

## Why open source it at all?

The purpose of open-sourcing the project is to preserve the codebase, project structure, and community-maintained development path so the project can survive beyond any one host or operator.

This helps with:

- continuity
- transparency
- community contribution
- preservation
- long-term maintainability

## Does this mean the official Ascendant Server is shutting down?

No.

Open-sourcing the repository does **not** mean the official server is shutting down.

It means the project is being structured so that its code, customizations, and core materials are not dependent on a single person or machine.

## Can someone make a fake Ascendant server and claim it is official?

Someone can create a fork or run a compatible instance, but that does **not** automatically make it official.

Only a server recognized by the project’s Community Stewards and operators should be considered an official Ascendant server.

If needed, the project can clarify publicly which live server is considered official.

## Will forks be allowed?

Yes, subject to the project license and governance model.

Forks are a normal part of open-source development. That said, not every fork is an official continuation or official live Ascendant server.

## Will community forks be supported?

That depends on the project stewards, available maintainers, and the nature of the fork.

The repository exists to preserve the project and allow broader continuity, but support for third-party forks may vary.

## Can other people contribute?

Yes.

Trusted maintainers and community contributors may contribute fixes, improvements, documentation, scripts, and other project changes through the repository process.

See `CONTRIBUTING.md` and `GOVERNANCE.md` for more details.

## Who controls the project?

The repository is overseen by a small group of **Community Stewards** who share responsibility for continuity, governance, and repository administration.

This structure is intended to prevent the project from depending entirely on one individual.

## Does the project still have a founder or lead operator?

Yes.

Straps was the original maintainer of Ascendant and continues to host a freely available live Ascendant instance.

At the same time, the repository is intended to be community-maintained and community-preserved over time, so continuity does not depend entirely on one person remaining in control forever.

## Why are some files missing?

Some files are intentionally excluded because they contain:

- secrets
- runtime-only artifacts
- private configuration values
- private operational data
- large generated files that do not belong in source control

This is intentional.

## Does this repository guarantee that anyone can run a perfect clone of the official server?

No.

The repository is intended to preserve the project and make compatible community-run instances possible, but it does not guarantee an exact clone of any live private production environment.

Some live operational details and private data are intentionally excluded.

## Is this replacing the current Ascendant community?

No.

This repository is meant to support and preserve the project, not replace the current live community.

## Why keep some things private if the project is open?

Because openness of the codebase is different from exposing private player data, production secrets, and sensitive operational records.

The goal is to preserve the project responsibly, not to dump every private live-server artifact into public version control.

## How should players think about this change?

Players should think of this as a long-term continuity and preservation step.

The goal is to make the Ascendant project more durable, more maintainable, and less dependent on one person or one machine, while still protecting private live-server information.

## How should outside observers think about this change?

Outside observers should view this repository as the community-maintained project home for Ascendant Server.

It is a preservation, continuity, and governance step — not a release of private player data and not a claim that every compatible instance is official.
